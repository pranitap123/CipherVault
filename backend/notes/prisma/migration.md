Question: what is a migration?
Answer: Right now your schema and your actual database disagree. The schema file says "there should be a files table with these columns." The database says "I have no such table." A migration is the bridge: it's a file of SQL commands (the CREATE TABLE files (...) statements) that transforms the database from its current state to match your schema. The reason this is a file and not just a live command is the important part — migrations get committed to Git, so every developer and every server can replay the exact same sequence of database changes in the exact same order. Your teammate clones the repo, runs the migrations, and their database is structurally identical to yours. That's why we don't just hand-write CREATE TABLE in a SQL console — the migration file is the versioned, shareable, repeatable record of how the database got its shape.
